Is using relative sizes, necessary to create a responsive webpage?

I agree with the opinion that relative sizes should be used necessarily when having in mind creating a respnsive and consistent website.
There are two kinds of measurement.

  • Absolute measurement
  • Relative measurement

Absolute measurement as its name states is an absolute value that does not depend on any other value. pixels for example. If am creating lets say a box, and I use pixel measurement size, for the height and width of that box, am simply telling the browser to give the box a height of this much of pixel and a width of this much of pixel. But what is a pixel? Well a pixel is a building block of the screen of our computer or phone or any other device that can support a browser. it is rectangular and very small in size, can not be seen by human eye. our devices ae made up of this rectangular elements. So pixels have fixed size that is decided during manufacturing. This means that once you state the amount of pixel, it is fixed.
Relative measurement also as it's name states, clearly depends on something and that, depending on the standard you are using can be any of, the browser's total width, or the box of the container holding it, or the default browser width and so on. This means unlike the absolute measurement that gives you discrete amount of blocks to build your house, you can now depending on the size of the house you want to build, can build one half, twice or any proportion of what you've got in mind. It gets even better, because now that you can give measurements relatively, no matter how large or small the screen gets, the shape or simply your measurement will sustain it's proportion which incase of building responsive websites is ideal condition.

How do browser's interprete the meaurements given to them?

Well Webpage's are not pictures or ready made files that can be downloaded and displayed, but rather they are instructions that tell our browsers to draw this picture for us to see. As you can guess this is not that easy after all who would paint websites in matter of seconds huh? So what the browser does is first create a DOM Tree (Document Object Model Tree) from the html file it gets. This DOM tree is a data structure in which there are nodes pointing to other nodes and etc. This nodes are html tags and this tree structure is an ideal place specially for files with lots of nested tags. And again our css file is there wondering how to apply all the stylings written in it to individual tags without any error. Again our browser stores this styling code in a tree structure. A different one. This tree structure is called CSSOM tree(cascading style sheet object model tree). Here also every styling attribute specific to a tag in the html file are stored as nodes. So when both tree are constructed our browser now merges these trees which now most probably is very easy for them to find one another. but when All these things happen there are defaults and standards that the browser always uses. One and important example is the rem.rem stands for root element it is a standard that our browser uses when sizing objects. It is more of a relative measurement since it is the default font size of the browser which is usually 16px. well if lets say our browsers default font size is 16px, then how many rem would a 200px size be?
1rem = 16px
X = 200px
which is equal to 12.5 rem. so based on this conversion mechanism and many other like precent to pixel and viceversa, percent to rem and vice versa and many other conversions, our browser understands them and draw them for us on our screen.

What are the specific changes brought to us by the new version of css, the css3?

Text Shadow

Text shadow property allows to add drop shadows to the text on a webpage. This is done by image or duplicating text element and then positioning it properly.

Box Shadow

It allows to create a drop shadow for an element. Most of the time this effect is done by using repeated image around the element, but because of box shadow that is done with a single line of code.

Border Images

It is the ability to swap out a border with an image. It allows to specify an image to display instead of a plain solid colored border.

Rounded Corners

Before css3 a lot of code was required to create a rounded corner. Give attention on height, width and positioning is very boring. CSS3 introduces border radius property which gives the same rounded corner effect.

Selectors

CSS3 introduce some new selectors. By using these selectors it is possible to choose DOM elements based on their attributes, so it might not necessary to specify classes and IDs for every element. Instead of that utilizing the attribute field to style them.

Web fonts

CSS3 introduces embedding any custom font on a webpage. Web pages render only fonts that are supported by the browser or client machine.

Multicolumn Layout

Most of webpages are made by columns or boxes and these boxes are adjusted by designers. Multicolumn layout simplify this problem and allow to specify only the number of columns and then they will be formed.

Transform

It enables to rotate web elements on a webpage. Before CSS3 JavaScript was required to rotate an element and they consume more resources, so CSS3 simplify this by introduce transform attribute.

RGBA

RGB is used to set colors for different elements, but now CSS3 introduces a modification and added opacity manipulation to this property. RGBA stands for Red Green Blue Alpha Channels.

Linear and radial gradients

In css3 we can add color gradients which are so amazing and very useful.
